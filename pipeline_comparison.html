🎯 Accuracy Issues — Local ML

  • ✓ Aspect classification is deterministic — no hallucinated aspects possible
  • ✓ Aspect-relative sentiment finds the RIGHT sentence for each aspect, not just overall tone
  • ⚡ Edge case: Sentence splitting regex may misfire on unusual punctuation (e.g., URLs, ellipses)
  • ⚡ Edge case: Short comments (<5 chars after split) fall back to comment-level sentiment
  • ✓ BERT sentiment model trained on 629K reviews — high domain fit for product feedback
  • ✓ Unmapped comments tracked — warns if >12% don't match any aspect

⚠️ Accuracy Issues — LLM-Only

  • ✗ Token truncation: Last comments in batch may be silently dropped when output exceeds max_tokens
  • ✗ Count mismatch: LLM returns 498/500 → entire analysis throws ValueError, no partial results
  • ✗ Aspect hallucination: LLM may assign aspects not in the frozen taxonomy
  • ⚡ Sentiment uniformity: "Great UX but terrible pricing" → both aspects get same sentiment
  • ✗ Scale degradation: Quality drops as batch size increases (500+ comments particularly bad)
  • ⚡ Retry cost: Failed batches add 2-8s delay each, compounding with multiple failures

Why Local ML Pipeline Wins

The Local ML pipeline processes 100% of comments deterministically with aspect-relative sentiment — something the LLM pipeline fundamentally can't do at scale. It uses 1 GPT call (narration only) vs 18+ for LLM-only, is 5-7× faster, costs 15-25× less per analysis, and never drops comments due to token truncation. The only trade-off: it needs ~1.5GB RAM for model weights (already handled by the B3 App Service Plan).

With CUDA on your RTX 4050: Local development latency drops from ~7 min to ~1-2 min for 50 comments, making the local ML pipeline viable for rapid iteration too.
